# Тестовое задание: Рефакторинг компонента админки

## Запуск проекта

```bash
npm install
npm run dev
```

По адресу http://localhost:8080/ откроется страница администрирования записей на курсы. 
Все запросы к АПИ замоканы (возвращаются рандомно сгенерированные данные), формат ответа сервера:

```javascript
{
  success: boolean,
  data: any,     // присутствует, если success === true
  message: string // присутствует, если success === false
}
```

## Задача

Вам предоставлен компонент `src/components/CourseRegistrations.vue`, 
который показывает список записей студентов на курсы и позволяет отменять регистрацию с указанием причины. 
Он написан в старом стиле (Options API) и содержит ряд проблем в коде и архитектуре.

**Нужно:**

1. Переписать компонент на Composition API (`<script setup>`).
2. Провести рефакторинг кода: улучшить структуру, читаемость и устранить неоптимальные решения, если обнаружены баги - исправить. Представьте, что это легаси-код, который вам нужно привести в порядок.
3. Сохранить весь текущий функционал:
   - загрузка списка из АПИ при монтировании компонента
   - поиск по имени/фамилии/email
   - фильтр по статусу
   - отмена записи через модальное окно подтверждения с возможностью указать причину отмены
   - отображение состояний загрузки и ошибки

**Обратите внимание**

Рекомендуемое время выполнения задания: 1 час.

Можно разбивать компонент на composables/подкомпоненты, выносить логику в функции.

Не нужно подключать дополнительные библиотеки или переписывать проект на другом стеке. 
Не нужно добавлять новый функционал (например, еще один фильтр) или существенно менять верстку/стилизацию.

Мы не ожидаем "идеального" решения с тестами и Storybook. 
Мы будем оценивать ход мыслей, внимание к деталям и качество итогового кода. 
Если что-то не успеете - опишите словами, что бы сделали дальше и почему.

## Формат сдачи

Пришлите архив с проектом (без `node_modules`) или ссылку на git-репозиторий.
